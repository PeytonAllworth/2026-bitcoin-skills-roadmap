**What to focus on:**
- The Cobb-Douglas formula: Y = A * L^Œ± * K^Œ≤
  - Y = output
  - L = labor input
  - K = capital input
  - A = technology/efficiency parameter
  - Œ±, Œ≤ = elasticities (how responsive output is to each input)
- What "returns to scale" means
- How to interpret the exponents (Œ± and Œ≤)

**Take notes on:**
- How could this apply to Bitcoin mining?
  - Y = hashrate produced?
  - L = labor (miners, operators)?
  - K = capital (ASIC machines)?
  - A = mining efficiency/technology?

### Week 3: Returns to Scale

**Assignment 3: Understanding Returns to Scale**
- üì∫ **[MRU: Returns to Scale](https://www.youtube.com/watch?v=8EEj5VHSFdw)** (6 minutes)
- üì∫ **[YouTube: Returns to Scale Explained with Examples](https://www.youtube.com/watch?v=OOuZJLlL4gQ)** (~10 minutes)
  - Search: "returns to scale economics" if link breaks

**What to focus on:**
- Constant returns to scale (double inputs ‚Üí double output)
- Increasing returns to scale (double inputs ‚Üí more than double output)
- Decreasing returns to scale (double inputs ‚Üí less than double output)
- In Cobb-Douglas: if Œ± + Œ≤ = 1, you have constant returns

**Think about:**
- Does Bitcoin mining have increasing or decreasing returns to scale?
- What happens when mining pools get bigger?
- Are there network effects or economies of scale?

## üìä Optional: Include Simple Analysis

If you want to go deeper, you can include:

**Simple Production Function Model:**
```python
import numpy as np
import matplotlib.pyplot as plt

# Simple Cobb-Douglas for Bitcoin Mining
# Hashrate = A * Capital^Œ± * Energy^Œ≤

A = 100  # Technology parameter (arbitrary units)
alpha = 0.6  # Capital elasticity
beta = 0.4  # Energy elasticity

capital = np.linspace(1, 100, 100)  # Capital (number of ASICs)
energy = 50  # Fixed energy input

hashrate = A * (capital ** alpha) * (energy ** beta)

plt.plot(capital, hashrate)
plt.xlabel('Capital (Number of ASICs)')
plt.ylabel('Hashrate Produced (TH/s)')
plt.title('Bitcoin Mining Production Function')
plt.grid(True)
plt.show()
```
